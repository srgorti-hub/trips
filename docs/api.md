# API / Data Interface — Travel Guide Generator

## Overview
This system has no HTTP API. The Travel Guide App is a static HTML file. The "API" is the structured data contract between Claude.ai (producer) and the Travel Guide App (consumer), defined by JSON schemas and file conventions.

The organizer uploads `trip-template.md` to Claude.ai, which contains these schemas. Claude generates the JSON files. The Travel Guide reads them at runtime.

### New Field: `location_keywords`

Present at both trip level and per-day level. Used by the Travel Guide App to fetch stock photos from Lorem Picsum when the organizer hasn't provided local photos.

- **Trip level** (`trip.location_keywords`): Used for the hero image. Should be broad terms describing the destination (e.g., `["Scottish Highlands", "West Highland Way"]`).
- **Day level** (`days[].location_keywords`): Used for day photos. Should be specific to the day's segment (e.g., `["Loch Lomond shore trail", "Conic Hill summit"]`).

Claude.ai generates these automatically based on the route research.

### Photo Fallback Chain

The Travel Guide resolves photos using a multi-tier fallback:
1. **Local file** — `photos/day-N/imgX.jpg` (organizer-provided)
2. **Wikimedia Commons search** — Searches by `location_keywords` for relevant photos
3. **Hide gracefully** — If no photo found

### YouTube Links

YouTube fields (`youtube_overview`, `days[].youtube`) accept either:
- **Search queries** (recommended) — e.g., `"West Highland Way hiking overview"` — the app converts to a YouTube search link
- **Full URLs** (legacy) — e.g., `"https://youtube.com/..."` — used as-is

### Interesting Links

`interesting_links[].url` is optional. If omitted, the app generates a Google search link from the `title`. This avoids broken links from hallucinated URLs.

### New Fields: `food_stops`, `toilet_facilities`, `interesting_links`

Per-day arrays providing practical information for hikers:
- **`food_stops`**: Restaurants, cafes, pubs, shops along the route with distance markers
- **`toilet_facilities`**: Public and commercial toilet locations with distance markers
- **`interesting_links`**: Points of interest (restaurants, historical sites, blogs, attractions) with URLs

### New Objects: `preparation`, `general_info`

Top-level objects providing trip-wide information:
- **`preparation`**: Training schedule, physical demands, and day-by-day expectations
- **`general_info`**: Visa info, emergency contacts, transport, currency, language, mobile coverage

### Validation Rules
- `trip.name`, `trip.destination`, `trip.description` must be non-empty strings
- `trip.dates.start` must be before `trip.dates.end`
- `trip.group_size` must be ≥ 1
- `trip.location_keywords` must have ≥ 1 entry
- `days` array must have ≥ 1 entry
- Each day's `day` field must be sequential starting from 1
- Each day's `date` must fall within `trip.dates.start` to `trip.dates.end`
- Each day's `location_keywords` must have ≥ 1 entry
- `difficulty` must be one of: `Easy`, `Moderate`, `Strenuous`
- `weather.precipitation_probability` must be between 0.0 and 1.0
- `gear_checklist[].essential` must be boolean
- All file paths (photos, maps, elevation_profile, gpx_download) are relative to trip root
- `food_stops[].type` must be one of: `restaurant`, `cafe`, `pub`, `shop`, `takeaway`
- `toilet_facilities[].type` must be one of: `public`, `cafe`, `pub`, `campsite`
- `interesting_links[].type` must be one of: `restaurant`, `history`, `attraction`, `blog`, `accommodation`, `other`
- `preparation.what_to_expect` should have one entry per day
- `general_info.emergency_contacts` should have at least the general emergency number

## Data Interface: `elevation/day-N-profile.json`

One file per day. Generated by Claude.ai. Consumed by the Travel Guide App to render elevation profile charts.

### Schema

```json
{
  "points": [
    {
      "km": "number — required — Distance from day start (km)",
      "elevation_m": "number — required — Elevation at this point (meters)"
    }
  ],
  "summit_label": "string — optional — Label for highest point (e.g., 'Conic Hill 361m')",
  "summit_km": "number — optional — Distance of summit from day start (km)"
}
```

### Rules
- `points` array must be sorted by `km` ascending
- First point should have `km: 0` (start of day)
- Last point should match approximately `days[N].distance_km`
- Points should be spaced ~0.5–2 km apart for smooth chart rendering
- `summit_km` should match the `km` of the highest `elevation_m` in `points`

## File Path Conventions

All paths in `trip-data.json` are **relative to the trip root folder**.

| Field | Example Value | Resolves To |
|-------|---------------|-------------|
| `days[0].photos[0]` | `"photos/day-1/img1.jpg"` | `/west-highland-way/photos/day-1/img1.jpg` |
| `days[0].map` | `"maps/day-1-map.jpg"` | `/west-highland-way/maps/day-1-map.jpg` |
| `days[0].elevation_profile` | `"elevation/day-1-profile.json"` | `/west-highland-way/elevation/day-1-profile.json` |
| `days[0].links.gpx_download` | `"gpx/day-1.gpx"` | `/west-highland-way/gpx/day-1.gpx` |

## localStorage Keys

Used by the Travel Guide App for client-side state persistence.

| Key | Type | Purpose |
|-----|------|---------|
| `gear-checked` | JSON array of strings | Item names the user has checked off |
| `theme` | `"light"` or `"dark"` | User's dark mode preference |
| `units` | `"metric"` or `"imperial"` | User's unit preference (km/m vs mi/ft) |

## Claude.ai → Travel Guide Handoff

1. Organizer uploads `trip-template.md` + itinerary to Claude.ai
2. Claude generates `trip-data.json` and `elevation/day-N-profile.json` files
3. Organizer downloads the files
4. Organizer places them in the trip folder alongside `index.html`
5. Optionally adds photos via `photos.html`, maps, and GPX files
6. Deploys folder to GitHub Pages or any static host

No direct communication between Claude.ai and the Travel Guide. The JSON files are the complete contract.
